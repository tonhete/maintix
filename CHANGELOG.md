## Formato de versiones

El proyecto seguirá el formato de versionado semántico: MAJOR.MINOR.PATCH

- **MAJOR:** Cambios incompatibles en la API
- **MINOR:** Nueva funcionalidad compatible
- **PATCH:** Corrección de bugs
